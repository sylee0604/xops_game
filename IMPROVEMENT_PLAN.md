## 1. 게임성 / 게임플레이 루프

### 현재 문제점

- **목표가 단순**: 모든 미션이 "적을 모두 제거하라" 하나. 승리/실패 조건 외에 아무것도 없음.
- **진행감 없음**: 미션 클리어 후 점수·등급·보상이 없음. 왜 잘 해야 하는지 동기부여가 없음.
- **난이도 조절 없음**: 적 스폰 수, 데미지, 사정거리가 고정. 초보자와 숙련자 경험이 동일.
- **재플레이 가치 없음**: 맵 레이아웃이 고정, 적 배치 고정, 픽업 위치 고정. 두 번 플레이하면 외워짐.
- **사망 페널티 없음**: 죽으면 그냥 게임 오버. 체크포인트나 리스폰 없음.
- **플레이어 성장 없음**: 무기 해금, 스킬 업그레이드, 어떤 형태의 성장도 없음.

### 개선 방안

```
A. 점수 시스템 도입
   - 적 처치: +100pt (헤드샷 +50pt 보너스)
   - 클리어 타임 보너스: 기준시간보다 빠를수록 +pt
   - 피격 없이 클리어: +500pt
   - 미션 종료 후 S/A/B/C 등급 표시

B. 난이도 선택 (Easy/Normal/Hard/Realism)
   - Easy: 적 HP -30%, 사정거리 -20%
   - Realism: HUD 없음, 아군사격 활성화, 원샷킬 적용

C. 체크포인트 시스템
   - 맵 중간에 체크포인트 존 설치
   - 사망 시 마지막 체크포인트에서 부활

D. 미션 브리핑 화면
   - 시작 전 목표·맵 미리보기·무기 선택 화면
```

## 2. 총기 평가

### 현재 무기 목록 및 문제점

| 무기 | 역할 | 문제 |
|------|------|------|
| Desert Eagle | 보조권총 | 보조무기 슬롯 개념 없어 사실상 잉여 |
| AK-47 | 주력소총 | 균형 잡혀 있음 |
| Knife | 근접 | 근접전 애니메이션 없이 즉시 데미지 적용 |
| AWP | 저격 | 조준경 전환 딜레이, 볼트액션 애니메이션 없음 |
| M14 EBR | 반자동저격 | AK와 차별점 부족 |
| MP5 | SMG | 포지셔닝이 AK보다 약한 버전 느낌 |
| SPAS-12 | 샷건 | 샷건치고 근거리 DPS가 낮음(pellet 7개 × 28 = 196 but fire rate 0.75) |
| M249 | LMG | ammo 100발인데 reload 3.2s로 너무 짧음 |

### 구체적 문제

1. **무기 교체 애니메이션 없음**: 무기 전환이 즉시 적용됨. 시각적 피드백 전무.
2. **재장전 애니메이션 없음**: `reloading` 플래그만 존재. 탄창 꺼내는 시각 없음.
3. **헤드샷 판정 없음**: 몸 어디를 쏴도 동일 데미지. 스킬 격차 없음.
4. **권총 슬롯 미구현**: DE는 carrySlots에 들어가지 않음. 주무기와 같은 슬롯 경쟁.
5. **총기 반동 표현**: pitch 반동만 있음. 수평(yaw) 반동 없음.
6. **총기음**: Web Audio 기반 필터만 적용. 실제 총성이 아닌 노이즈 합성. 총기마다 특성 차이가 미묘함.

### 개선 방안

```
A. 추가 무기 (최소 2종)
   - 권총 슬롯 전용: Glock 17 (빠른 연사, 낮은 데미지)
   - 유탄발사기 또는 RPG: 특수 무기 슬롯
   - 연막/섬광탄: 전술 다변화

B. 헤드샷 시스템
   - 적 head mesh에 hitbox 분리
   - 헤드샷: 즉시 사망 또는 데미지 ×2.5
   - 화면에 "HEADSHOT" 텍스트 표시

C. 무기 교체 딜레이 + 애니메이션
   - 무기 아래로 내려갔다 올라오는 Y축 트윈 애니메이션
   - 교체 중 사격 불가 (0.4s 딜레이)

D. 수평 반동 패턴
   - AK: 약간 우측으로 퍼짐
   - MP5: 좌우 교번
```

## 3. 미션 종류

### 현재 미션 목록

| 미션 | 목표 | 특이점 |
|------|------|--------|
| Mission 1 - Compound | 적 전멸 | 실내 맵 |
| Mission 2 - Harbor | 적 전멸 | 항구, 낙사 가능 |
| Mission 3 - Tunnel | 좀비 전멸 | 좀비 돌진 |
| Mission 4 - Assault | 적 전멸 (아군 있음) | 아군 AI |
| Mission 5 - Rescue | 인질 3명 구출 | 발각 시 인질 사살 |
| Mission 6 - Defuse | 폭탄 3개 해제 | 90초 타이머 각각 |
| Survival | 무한 웨이브 생존 | 웨이브마다 난이도 상승 |
| Training | 사격 연습 | 적 없음 |

### 문제점

- ~~**모든 미션 목표가 동일**~~ → ✅ 인질구출(RESCUE), 폭탄해제(DEFUSE), 무한생존(SURVIVAL) 추가됨
- **미션 연결성 없음**: 미션 1 클리어해도 2가 잠금해제되지 않음. 선택 화면에서 다 열려 있음.
- ~~**특수 목표 없음**~~ → ✅ 구현됨

### 개선 방안

```
A. 미션 목표 다양화 (필수)

   Mission 5 — RESCUE
   - 건물 내부에 인질 3명 포로로 잡혀 있음
   - 적에게 발각되면 인질 즉시 사살 → 실패
   - 스텔스 or 빠른 제압 필요

   Mission 6 — DEFUSE
   - 맵 3곳에 폭탄 설치됨
   - 각 폭탄 90초 타이머
   - 플레이어가 폭탄 위치에 3초 머물면 해제
   - 하나라도 폭발하면 실패

   Mission 7 — ESCORT
   - VIP NPC를 출구까지 에스코트
   - VIP는 자동으로 이동, HP 보유
   - 적이 VIP에게 사격

   Mission 8 — SURVIVE
   - 무한 웨이브 생존 모드
   - 웨이브마다 적 수 +2, 속도 +10%
   - 최대 웨이브 기록 갱신 방식

B. 미션 선택 화면 개선
   - 순차 잠금 해제 시스템
   - 각 미션 클리어 등급 표시
   - 베스트 타임 표시
```

## 4. 맵 다양성

### 현재 맵 문제점

- **맵 4개 중 3개가 평지**: Compound는 격자형 실내, Harbor는 부두, Assault는 전쟁터. 수직감 거의 없음 (2층 구조 1개).
- **동적 요소 없음**: 문이 열리거나, 엄폐물이 파괴되거나, 조명이 변하는 일이 없음.
- **환경 상호작용 없음**: 스위치, 버튼, 이동하는 물체 없음.
- **맵 크기가 작음**: 가장 큰 Assault 맵이 39×75 유닛. 이동이 금방 끝남.
- **탈출구/진입로 다양성 없음**: 대부분 직선 경로.

### 개선 방안

```
A. 새 맵 2개 추가

   맵 5 — ROOFTOP (도심 옥상)
   - 여러 건물 사이 점프 경로
   - 저격 포지션 다수
   - 헬기 소음 ambient 효과

   맵 6 — UNDERGROUND (지하 시설)
   - 좁은 통로 + 넓은 중앙 홀 교차
   - 조명 깜빡임 이벤트 (일시적 어둠)
   - 전기 박스 상호작용으로 구역 잠금/해제

B. 기존 맵 개선
   - Compound: 문 개폐 메커닉 추가
   - Harbor: 크레인 조작 가능 (엄폐물 이동)
   - Tunnel: 조명 터지는 이벤트 → 플래시라이트 필요성

C. 파괴 가능 오브젝트
   - 나무 상자(crate)에 일정 피탄 시 파괴
   - 유리창 파괴
```

## 5. 그래픽

### 현재 문제점

1. **박스 폴리곤 모델**: 적, 무기 모두 BoxGeometry 조합. 실루엣이 딱딱함.
2. **파티클 효과 없음**: 총구 화염(muzzle flash)은 PointLight만 있음. 입자 없음. 피격 시 스파크/피 없음.
3. **피격 이펙트 없음**: 적이 맞아도 변화 없음. 사망 시 그냥 쓰러짐 (y축 회전).
4. **환경 텍스처 없음**: 모든 표면이 단색 Material. 콘크리트, 나무, 금속 구분 없음.
5. **그림자 품질 낮음**: PCFShadowMap 사용 중, 그림자 해상도와 범위 제한적.
6. **탄흔(Bullet Decal) 없음**: 벽에 맞아도 임팩트 스파크만 있고 탄흔이 남지 않음.
7. **사망 애니메이션 빈약**: 적 사망 시 회전만 함. ragdoll 없음.
8. **HUD가 기초적**: 체력바, 탄약, 미니맵만 있음. 데미지 방향 표시 없음.

### 개선 방안

```
A. 파티클 시스템 구현 (우선순위 높음)
   - THREE.Points 기반 파티클 풀 (최대 200개)
   - 총구 화염: 0.04s 지속 노란/흰 파티클 3~5개
   - 피격 스파크: 콘크리트 타격 → 회색 파티클
   - 혈흔 이펙트: 적 피격 → 붉은 파티클 (옵션 설정 가능)

B. 탄흔 시스템
   - 벽 타격 시 DecalGeometry 또는 작은 plane mesh 부착
   - 최대 50개, 오래된 것부터 제거

C. HUD 개선
   - 데미지 방향 인디케이터 (화면 가장자리 붉은 플래시)
   - 킬 피드 (우측 상단: "플레이어 → 적" 표시)
   - 클리어 카운터 (남은 적 수 / 전체 수)

D. 텍스처 도입 (선택적)
   - canvas로 절차적 텍스처 생성 (노이즈 기반 콘크리트, 나무결)
   - Three.js CanvasTexture 활용으로 외부 파일 의존성 없이 구현 가능

E. 피격 효과
   - 체력 30% 이하: 화면 가장자리 빨간 비네팅
   - 피격 시 화면 순간 적색 플래시
```

## 6. 조작감

### 현재 문제점

1. **점프 느낌**: JUMP_FORCE=4.9, GRAVITY=-18. 약간 둔하고 아케이드적임.
2. **bhop 시스템**: 구현돼 있으나 일반 플레이어에게 설명 없음. 발견하기 어려움.
3. **ADS(조준경)**: 마우스 우클릭 → 즉시 FOV 변경. 전환 트윈 없음. 급격함.
4. **크로스헤어 없음**: 기본 조준점이 없음. HUD 중앙 점이 있긴 하나 너무 작음.
5. ~~**발소리 없음**~~ → ✅ **구현 완료 (2026-04-02)**: 플레이어 달리기/크라우치 발소리, 적 15유닛 내 거리 기반 발소리. AI 인식 연동 없음.
6. **무기 밥(Bob) 애니메이션**: 이동 시 무기 흔들림이 없음. 정지한 것처럼 보임.
7. **무기 관성 없음**: 마우스를 빠르게 돌려도 무기가 즉시 따라옴. 중량감 없음.
8. **벽 충돌 미끄러짐**: 대각선 이동 시 벽에 걸리면 멈추는 경우 있음.

### 개선 방안

```
A. ADS 전환 트윈 (0.1s 보간)
   - FOV를 즉시 바꾸지 않고 lerp로 서서히 변경
   - 무기 viewPos도 ADS 위치로 lerp 이동

B. 무기 밥(Bob) 구현
   - 이동 시 sin 파형으로 무기 Y/X 오프셋 가산
   - 달릴 때 진폭 증가, 멈출 때 감쇠

C. 무기 관성 (Sway)
   - 마우스 이동 델타에 비례한 무기 회전 오프셋
   - 시간이 지날수록 원위치로 복귀

D. 발소리 시스템
   - Web Audio로 발소리 주기적 재생 (이동 중일 때)
   - 크라우치 시 소리 줄어듦
   - 적 발소리 (일정 거리 이하 시 재생)

E. 크로스헤어 개선
   - 동적 크로스헤어: 이동/사격 시 벌어짐
   - 적 조준 시 색상 변경 (빨간색)
```

## 7. 최적화

### 현재 문제점

1. ~~**Draw Call 과다**~~: → ✅ **InstancedMesh 전환으로 해결 (2026-04-02)**
   - SOLDIER(50)/ZOMBIE(30)/ALLY(8) 인스턴스 풀, 부위별 InstancedMesh
   - 적 50명 기준 ~55 draw call 고정 (이전: 200+ per 10명)
   - 공유 white 재질 + instanceColor로 per-unit 색상 제어

2. **Shadow 불필요한 오브젝트도 castShadow 가능성**: 모든 mesh에 shadow 설정 확인 필요.

3. **wallMeshList 레이캐스트**: 매 프레임 모든 적이 wallMeshList 전체를 레이캐스트. 적 수 증가 시 O(n×m) 부하.

4. **픽업 updatePickups()**: 매 프레임 전체 픽업을 순회하며 거리 계산. 공간 분할 없음.

5. **Geometry 재생성**: `addSandbagWall`에서 루프 내에 `new THREE.BoxGeometry()` 호출. 합칠 수 있음.

6. **메모리 누수 가능성**: 게임 재시작 시 scene을 새로 만들지만 이전 geometry/material dispose 호출 없음.

### 개선 방안

```
A. ✅ 적 모델 InstancedMesh 전환 (완료 2026-04-02)
```

## 8. 보안성

### 현재 문제점 (브라우저 게임 기준)

1. **전역 스코프 완전 노출**:
   ```js
   // 브라우저 콘솔에서 즉시 실행 가능:
   player.health = 999;
   player.weapons[1].ammo = 9999;
   enemies.forEach(e => e.health = 0);
   ```
   - `player`, `enemies`, `bullets`, `scene` 등 모든 핵심 상태가 `window` 객체에 노출됨.

2. **치트 방지 없음**: FPS 고정 없음 → 고FPS 환경에서 bhop 속도 이득 가능.

3. **입력 검증 없음**: `keys` 객체는 외부에서 직접 조작 가능.

4. **점수 조작**: 점수 시스템 추가 시 클라이언트 사이드만 존재하면 무의미.

5. **ES 모듈 미사용**: `'use strict'`는 각 파일에 있지만 `type="module"` 미사용으로 격리 없음.

### 개선 방안

```
A. IIFE 또는 ES 모듈로 전환 (핵심)
   - 각 js 파일을 IIFE로 감싸거나 type="module"로 변환
   - 외부 노출이 필요한 함수만 명시적 export
   - 전역 오염 제거만으로도 콘솔 치트 80% 차단

B. 클로저로 상태 캡슐화
   - player, enemies 등을 getter/setter로만 접근
   - 직접 속성 변경 시 유효성 검증

C. FPS 독립 속도 확인
   - bhop 최대 속도(BHOP_MAX_SPEED) 강제 클램프 재확인
   - dt 상한(현재 0.05s = 20FPS 최소) 적절함

D. 점수 서버 검증 (점수 시스템 추가 시)
   - 로컬 서버(서버시작.bat 활용)로 점수를 POST
   - 클라이언트에서 최종 점수 직접 변경 불가 구조로 설계

참고: 싱글플레이 로컬 게임이므로 보안 완벽화보다는
"실수로 콘솔 치트" 방지 수준이 현실적 목표임.
```

## 9. 코드 확장성

### 현재 장점

- `WEAPON_DEFS[]` 중앙 관리: 무기 추가 시 배열 항목 하나만 추가
- `spawnEnemyAt(x,z,wps,isZombie,floorY)` 파라미터화됨
- `js/maps/` 디렉토리로 맵 파일 분리됨

### 현재 한계

1. **전역 의존성**: 모든 파일이 `player`, `scene`, `enemies`, `walls` 등 전역 변수를 암묵적으로 참조. 파일 하나만 봐서는 의존성 파악 불가.

2. **맵 추가 시 game.js 수정 필수**: `startGame()` 내부에 `if/else if` 체인이 있음. 맵 추가마다 이 함수를 수정해야 함.

3. **미션 시스템 없음**: 승리 조건이 `enemies.length === 0` 하나. 복합 조건 추가 불가 구조.

4. **이벤트 시스템 없음**: "적 전멸 시 문 열림" 같은 트리거를 구현하려면 game.js 직접 수정 필요.

5. **상태 관리 단편적**: `gameStarted`, `gamePaused`, `currentMap` 등 게임 상태 변수들이 state.js에 산재. 게임 상태 전환 로직 없음.

### 개선 방안

```
A. 맵 레지스트리 패턴 도입
   // game.js의 if/else if 대신:
   const MAP_REGISTRY = {
     combat:   { build: buildMap, spawn: spawnEnemies, ... },
     harbor:   { build: buildHarborMap, spawn: spawnHarborEnemies, ... },
     assault:  { build: buildAssaultMap, spawn: spawnAssaultEnemies, ... },
   };
   // 맵 추가 = registry에 항목 추가만 하면 됨

B. 미션 조건 객체화
   // 현재: if (enemies.filter(e=>e.health>0).length===0) showVictory()
   // 개선:
   const missionGoal = {
     type: 'eliminate_all',  // 'defuse', 'escort', 'survive'
     check: () => enemies.every(e => e.health <= 0),
     onSuccess: () => showVictory(),
   };

C. 이벤트 버스 (간단 구현)
   // js/events.js 추가
   const GameEvents = {
     _listeners: {},
     on(ev, fn)   { (this._listeners[ev] ??= []).push(fn); },
     emit(ev, d)  { (this._listeners[ev] ?? []).forEach(fn => fn(d)); },
   };
   // 사용: GameEvents.on('enemy_died', () => checkMission());

D. 의존성 명시화 (장기)
   - type="module" 전환 후 import/export 명시
   - 각 모듈이 어떤 상태를 읽고 쓰는지 명확히
```
